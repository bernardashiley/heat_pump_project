# HeatpumpMonitor Evaluation Data

This directory holds the inputs and outputs for the v1 calibration evaluation against HeatpumpMonitor.org. Most files in this directory are not committed to the repository for either licensing or regenerability reasons.

## What is committed

- `climate_cache/` — 306 parquet files of historical UK daily mean outdoor temperature from Open-Meteo, one per unique 4-decimal-degree latitude/longitude pair in the evaluation set. Each file contains 20 winters of October–March daily mean temperatures. Open-Meteo data is licensed under CC BY 4.0 with attribution; see https://open-meteo.com/en/license.

- This README.

## What is not committed

- The HeatpumpMonitor.org snapshot JSON files (`systems_*.json`, `stats_all_*.json`). The HeatpumpMonitor.org dataset is openly available via the project's public API but does not carry an explicit dataset licence permitting redistribution. The fetch script `backend/scripts/fetch_heatpumpmonitor.py` retrieves the same data directly from the source.

- The derived evaluation dataset `eval_cases.json` and filter report `eval_filter_report.txt`. These are regenerated by `backend/scripts/build_eval_dataset.py` from the HeatpumpMonitor fetch.

- The evaluation outputs `calibration_predictions.json`, `calibration_errors.csv`, `calibration_report.md`. These are regenerated by `backend/scripts/run_calibration_eval.py`.

## To reproduce the v1 evaluation

From a clean checkout of the repository, with the Python environment set up per the top-level README:

```powershell
backend.venv\Scripts\python.exe backend\scripts\fetch_heatpumpmonitor.py
backend.venv\Scripts\python.exe backend\scripts\build_eval_dataset.py
backend.venv\Scripts\python.exe backend\scripts\run_calibration_eval.py
```

The fetch step is non-deterministic because the HeatpumpMonitor.org dataset grows over time: systems are added and removed, stats are updated. A snapshot taken now will not exactly match the snapshot of 9 June 2026 used to produce the v1 calibration report. To produce numerically identical results to the published v1 report, the original snapshot files would be needed; they were committed at hash 0301661 in the gitignored state and are no longer recoverable from the public repository. For an exact-replication run, contact the author (see the report's section 12.1).

The climate cache committed here is reusable across any HeatpumpMonitor snapshot that overlaps in monitoring locations. The evaluation script will fetch additional climate data for any new locations not in the cache; the prefetcher script `prefetch_climate_cache.py` is available for controlled population.
